# S00ubifs — защита флешки и RAM-режим

Почему используется tmpfs и где это НЕ нужно включать.

---

## Задача

Снизить износ флешки (/opt) за счёт переноса временных данных в RAM.

---

## Проблема

Entware активно пишет:
- логи (mihomo, watchdog)
- PID-файлы
- временные файлы
- кэши

При постоянной работе:

👉 флешка умирает через 6–12 месяцев  
👉 особенно на 20+ роутерах  

---

## Решение

Перенести volatile-данные в RAM:

```

/opt/tmp
/opt/var/log
/opt/var/run

```

через tmpfs

---

## Как это работает

Сервис:

```

/opt/etc/init.d/S00ubifs

````

Запускается раньше всех (`S00`).

---

### При старте

```bash
mount -t tmpfs -o size=20M tmpfs /opt/tmp
mount -t tmpfs -o size=10M tmpfs /opt/var/log
mount -t tmpfs -o size=1M  tmpfs /opt/var/run
````

---

### Что остаётся на диске

* /opt/bin (пакеты)
* /opt/etc (конфиги)
* config.yaml
* crontab

---

### Что уходит в RAM

* логи
* временные файлы
* сокеты
* PID

---

## Почему это важно

Без этого:

* постоянная запись на флешку
* деградация скорости
* eventual failure (смерть накопителя)

---

## Главный момент: ОГРАНИЧЕНИЕ ПО RAM

👉 **Минимум: 256 МБ RAM**

---

## ❌ НЕ ПОДДЕРЖИВАЕТСЯ (реальный опыт)

### 128 МБ устройства

Примеры:

* Keenetic Extra (MT7628)
* Keenetic Viva старый (MT7621, 128MB)

---

### Что происходит

* RAM занята почти полностью (~120/128)
* не стартуют штатные сервисы Keenetic
* нестабильная работа
* random падения

👉 система начинает "сыпаться"

---

### Вывод

❌ **НЕ использовать на 128MB устройствах**

Это не рекомендация — это проверено в продакшене.

---

## ✔️ Рекомендуется

| RAM     | Статус                  |
| ------- | ----------------------- |
| 128 MB  | ❌ нельзя                |
| 256 MB  | ⚠️ минимально допустимо |
| 512 MB  | ✅ нормально             |
| 1024 MB | ✅ идеально              |

---

## Почему именно такие размеры tmpfs

| Директория   | Размер | Почему          |
| ------------ | ------ | --------------- |
| /opt/tmp     | 20M    | временные файлы |
| /opt/var/log | 10M    | логи            |
| /opt/var/run | 1M     | PID и сокеты    |

---

## Проверка

### Смонтировано ли

```bash
mount | grep tmpfs
```

---

### Использование RAM

```bash
df -h /opt/var
```

---

### Статус сервиса

```bash
/opt/etc/init.d/S00ubifs status
```

---

## Что теряется при reboot

Это важно понимать:

* логи watchdog
* логи mihomo
* временные файлы

👉 это НОРМАЛЬНО

---

## Когда использовать disk режим

Если:

* мало RAM
* нужен persistent лог
* нестабильность системы

👉 отключить tmpfs

---

## Почему не весь /opt в RAM

Потому что:

* пакеты должны сохраняться
* конфиги должны сохраняться
* иначе после reboot всё исчезнет

---

## Почему именно init.d (S00)

* запускается раньше всех сервисов
* гарантирует, что:

  * cron
  * mihomo
  * watchdog

уже работают с tmpfs

---

## Частые проблемы

### Не смонтировалось

Причины:

* мало RAM
* ошибка mount

Проверка:

```bash
dmesg | grep tmpfs
```

---

### Система стала нестабильной

👉 почти всегда нехватка RAM

Решение:

* отключить S00ubifs
* или перейти на роутер с 256+ МБ

---

## Итог

S00ubifs:

✔ защищает флешку
✔ снижает запись
✔ повышает ресурс

НО:

❗ требует достаточной RAM
❗ не работает на слабых устройствах

---

## Коротко

👉 Есть RAM → включаем
👉 Нет RAM → даже не пробуем

```
